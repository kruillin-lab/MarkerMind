# Stage 01: Blueprint

**Layer 2 Context - BLAST Protocol (B)**
This stage is responsible for research, task discovery, requirement mapping, and creating an actionable implementation plan.

## Inputs
| Source | File/Location | Section/Scope | Why |
|--------|--------------|---------------|-----|
| User Request | Ongoing session | Current chat | Core requirements |
| Context Rules | [[../../_config/brand-guidelines\|Brand Guidelines]] | General | Aesthetic/voice matching |
| Active Tasks | [[../../MEMORY\|MEMORY.md]] | Active items | What are we already doing |

## Process
1. Evaluate the user's intent.
2. Formulate a blueprint or research summary.
3. Identify constraints and requirements.
4. **Checkpoint**: Ask the user to approve the scope before writing the output.
5. Create an implementation plan formatting it with Obsidian Wikilinks where applicable.
6. Write the final blueprint to `output/blueprint.md`.

## Outputs
| Artifact | Location | Format |
|----------|----------|--------|
| Blueprint | [[output/blueprint\|output/blueprint.md]] | Obsidian Markdown |
