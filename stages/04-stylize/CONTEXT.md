# Stage 04: Stylize

**Layer 2 Context - BLAST Protocol (S)**
This stage handles refactoring, aesthetics, documentation standards, and overall cleanup. It ensures that the Architect's output meets the brand voice, UI/UX, or code stylistic requirements.

## Inputs
| Source | File/Location | Section/Scope | Why |
|--------|--------------|---------------|-----|
| Architect Log | [[../03-architect/output/architect-log\|03-architect/output/architect-log.md]] | Full file | Actionable scope of built code |
| Brand Guidelines | [[../../_config/brand-guidelines\|Brand Guidelines]] | General | Aesthetic/voice matching |
| Voice Rules | [[../../_config/voice-rules\|Voice Rules]] | General | Writing constraints |

## Process
1. Read the architect implementations.
2. Verify formatting (e.g. Obsidian Wikilinks, Kebab-case).
3. Check UI designs or naming conventions.
4. Add comments, polish code aesthetics, and organize formatting.
5. Record changes in `output/stylize-log.md`.

## Outputs
| Artifact | Location | Format |
|----------|----------|--------|
| Stylize Log | [[output/stylize-log\|output/stylize-log.md]] | Obsidian Markdown |
| Formatted Files | Vault/Git | Native Code/Files |
